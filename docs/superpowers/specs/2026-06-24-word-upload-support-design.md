# Word 上传支持设计

## 背景

原 MVP 仅支持可复制文本型 PDF。实际使用中，专利申请文件常以 Word 文档流转，同时附图说明并非每个申请都一定存在，因此上传工作台需要支持 Word 文件，并允许用户不上传附图说明。

## 范围

- 权利要求书、说明书支持上传可复制文本型 PDF 或 Word（`.docx`）。
- 附图说明文件从必填改为可选；摘要文件继续可选。
- 附图说明和摘要如上传，同样支持 PDF 或 `.docx`。
- 旧版 `.doc` 暂不作为承诺格式，因为生产容器内缺少可靠的纯 Python 解析链路。

## 后端设计

后端新增统一文档抽取入口 `document_extractor`，按文件后缀分发：

- `.pdf`：沿用 `pdf_extractor`，只处理可复制文本型 PDF，扫描件继续提示暂不支持 OCR。
- `.docx`：使用 `python-docx` 抽取段落与表格文本，空文档或损坏文档返回用户可读错误。

任务创建服务的必填角色只保留 `claims` 和 `specification`。`drawings` 与 `abstract` 若未上传，则对应文本长度为 `0`，模型第二阶段仍接收空字符串并按已有提示处理。

## 前端设计

工作台文件选择器的 `accept` 放开为 PDF 与 `.docx`，附图说明字段移除必填标记。页面提示文案同步说明“支持可复制文本型 PDF 和 Word（.docx）”，并继续强调扫描件 PDF 不支持 OCR。

## 验证

- 后端接口测试覆盖 `.docx` 上传可创建任务。
- 后端接口测试覆盖不上传附图说明也可创建任务。
- 抽取层测试覆盖 `.docx` 段落文本读取。
- 前端构建验证上传页类型与文案变更不破坏编译。
